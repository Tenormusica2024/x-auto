# AI臭い表現の禁止ルール（統合版）

AI生成テキストの不自然さを排除するためのルール。
全ツイート種別で適用。

## 0. 根幹原則: 自分フィルター（全ルールの上位原則）

**すべての発言は「自分の判断を通過した意見」として表現する。「客観的事実の宣言」にしない。**

詳細は `common/expression-rules.md` の「自分フィルター原則」を参照。
この原則に照らして判断に迷ったら: 「これは神の視点から事実を宣言してないか？」と自問する。

## 1. 語尾・口調の禁止パターン

| 禁止パターン | 理由 |
|-------------|------|
| 「〜んだな」 | 今気づいた感。AIが新情報にリアクションしてる感じ |
| 「〜だよな」「〜だな」 | 批評家っぽい。上から評価してる感じ |
| 「〜なんだよね」（裸の断定） | 自分フィルターなし。分かってる人が教えてる感。「〜と思うんだよね」なら可 |
| 「〜かな...」 | ふわっと終わる。結論が出てない印象 |
| 「〜だよなー」「〜かなー」 | 伸ばし棒語尾。作為的なカジュアルさ |
| 「〜ですね」「〜でしょう」「〜だと思います」 | AI臭い丁寧語・敬語（※politeモード時は許可。tone.json参照） |

## 2. 接続・構文の禁止パターン

| 禁止パターン | 理由 |
|-------------|------|
| 「つまり」「逆に言えば」 | 論理的すぎる接続。人間は感覚で繋ぐ |
| 造語（例: 真面目期間、認知負荷） | AIは概念に名前をつけたがる。人間は既存の言葉の組み合わせで済ませる |
| レトリカルクエスチョン（「なぜか?」） | 自問自答型の演説調・インフルエンサー構文 |
| 対比構造+毎回改行 | 「Aか / Bか / Cだ」はビジネス書の煽り文句風 |

## 3. トーン・姿勢の禁止パターン

| 禁止パターン | 理由 |
|-------------|------|
| 表面的感想（「すごい」「驚いた」だけ） | 中身がない。AIの定型リアクション |
| 感情的評価語（「ヤバい」「えぐい」） | 若者言葉の不自然な使用 |
| 「新鮮なリアクション」の演出 | AIは初めて聞いた風にリアクションする。人間は「前から思ってた」トーンで話す |
| 相手に合わせた距離調整 | AIは共感レベルを調整しようとする。一定の距離感を保つ |
| メタ評価（「この発想が素晴らしい」等） | 批評家視点で上から評価してる感じ |
| 人物評価（「〇〇さんは先見の明がある」等） | 人を評価する立場にない |

## 4. 表現スタイルの禁止パターン

| 禁止パターン | 理由 |
|-------------|------|
| 詩的・哲学的表現（「〜と引き換えに〜」等） | 技術を美化・ロマン化してる。日本エンジニア界隈で嫌がられる |
| 宣伝・共有アピール（「参考になる」「〜も納得ですね」） | 「共有してます」感が嫌がられる |
| 英文翻訳感（「オーケストレーション税」等） | カタカナ造語・直訳風表現は受けが悪い |
| リスク懸念の過剰言及（毎回「〜に注意が必要」） | 定型文化して価値が薄れる |
| 当たり前の結論（「継続が大事」「使いこなしの深さが効く」） | 誰でも言える。読者に新しい気づきがない |
| 聴衆への訴えかけ（「〜してみよう」「みんなどんな工夫してる?」） | 弱小アカウントでは不自然。「自分は〜してる」で淡々と |
| 読点の多用 | プレゼン的。強調したいわけじゃない場合は省略 |

## 5. 自動化パイプラインの露出制御

ツイート生成・レビュー・投稿の自動化フローそのものを匂わせない。最終目標は「人間が書いたとしか思えないツイートを自動生成する」こと。

### 露出OKライン（情報収集・ツール利用のナレッジシェア）
- X APIで情報収集していること
- AIに指示して作業を効率化していること（具体的なツール名+指示内容を含める）
- コスト最適化・自動化の知見をナレッジとして共有すること

### 露出NGライン（生成パイプラインの存在を示唆するもの）
- ツイート文の自動生成・AI生成・文章調整を匂わせる表現
- リアクション分析・エンゲージメント自動調査を匂わせる表現
- レビュー・品質判定の自動化を匂わせる表現
- 「このツイートも自動で作ってるのでは？」と読者に疑わせる文脈

### 判定の実例
```
❌「X APIの日次スクリプトを見直した」
  → 「日次スクリプト」が具体的すぎて自動化パイプラインの存在を示唆

✅「最新トピックの情報調査用にX APIの日次スクリプトを見直した」
  → 「情報調査用」で目的を明示。嘘は言っていないが露出を制御している

❌「重複クエリ除去とバッチ化で$2.33→$0.27になった」
  → 「Claudeに指示した」がない。誰がやったかの具体感がなく抽象的

✅「重複クエリ除去とバッチ化をClaudeに指示したら$2.33→$0.27になった」
  → 行動主体を明示。読者に「自分もやってみよう」と思わせる具体性
```

### チェック粒度: 単語ではなくトピックで判定する

露出制御は**表現の言い換え**では解決しない。**トピック自体**がNGラインに該当する場合、どう書き換えてもNG。

```
❌ 単語だけ変えてトピックがNG
  「自動評価スクリプト」→「X APIで引っ張った」に変えても
  トピックが「自分のツイートのパフォーマンス分析」ならNG

❌ ツール名をOKラインのものに差し替えてもNG
  「W-Score上位」→「反応が厚い」に変えても
  「どのツイート種別がエンゲージメント高いか分析した」というトピック自体がNG

✅ トピック自体をOKラインに変える
  自分のツイート分析 → 別トピック（AI運用コスト・ツール比較等）に変更
```

**チェック手順:**
1. まず**トピック全体**が露出NGに該当しないか確認
2. NGならトピック自体を変更（表現の言い換えではなく）
3. OKなら個別の表現を確認

### 原則
- **嘘は言わない。ただし全容は見せない** — 情報収集の自動化は共有するが、文章生成の自動化は見せない
- **ナレッジシェアの価値を落とさない** — 露出を制御するために情報を抽象化しすぎると読者への価値が下がる。具体的な数値・手順・ツール名は維持する

## 6. 元ツイート関連の禁止（引用RT固有）

引用RTコメント生成時のみ追加適用:
- 元ツイートの情報をそのまま繰り返さない
- 元ツイートの主張をオウム返ししない
- AI目線の第三者的コメント禁止（「この取り組みは〜」）
- 三人称表現禁止（「自分もそう」と当事者視点で書く）
