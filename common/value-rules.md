# 投稿価値ルール（信頼残高基準）

全ツイート種別で適用。投稿の「価値」を判定する最上位基準。
出典: @shinkaron「毎月平均5000フォロワーづつ増やしてるけど、耳の痛い話をしてやんよ」

## 根幹原則: 信頼残高

フォロワー数は積み上げてきた信頼の残高。各投稿は信頼残高の増減に直結する個々の取引。
この前提を持たずに投稿を量産しても、長期的な成果にはつながらない。

## 価値提供の5条件（1つ以上を満たすこと）

投稿が「読み手の時間を投資してよかった」と思わせるには、以下のいずれかを達成する必要がある:

| 条件 | 具体例 |
|------|--------|
| **思考が一段深まった** | 既知の話題に新しい視点・切り口を提示 |
| **感情を大きく衝き動かされた** | 驚き・共感・危機感など感情の動きがある |
| **行動のきっかけを与えられた** | 「試してみよう」「調べてみよう」と思わせる |
| **パラダイムシフトを起こされた** | 前提を覆す、常識の再定義 |
| **有用な知識やノウハウが提供された** | 具体的な手順・数値・比較・ツール紹介 |

**5条件のいずれも満たさない投稿は信頼残高を減らす。投稿しない方がマシ。**

## 時間責任の原則

「一投稿ごとに、他者の時間と注意力に対して責任を負え」

読み手はフォロワーの有限な時間を使ってその投稿を読んでいる。
時間を浪費させられたと感じさせる投稿は、信頼残高を削る。

## 逆算設計

読者心理から逆算してポストを設計する。

```
❌ 思いつき → 投稿 → 反応を祈る
✅ 読者が得る価値を定義 → その価値を最大化する構成を設計 → 投稿
```

「このツイートを読んだ人は何を得るか？」を投稿前に必ず自問する。

## 人格投影の原則

発信者の人格・知識・能力・経験・実績をいかに投影させるかが決定的に重要。
誰が書いても同じ内容（一般論の要約、ニュースの転載）では信頼残高は増えない。

persona-dbの情報を活用し、以下を投稿に反映する:
- **経験**: 実際に使った・試した・感じたことベースで書く
- **知識**: 表面的な紹介ではなく、理解の深さが伝わる言及をする
- **スタンス**: 中立的な情報整理ではなく、自分の立場を明確にする

## 読者アクション起点の具体性原則

ナレッジシェア系のツイートでは、読者が「自分もやってみよう」「自分も気をつけよう」と思える具体性を維持する。露出制御やプライバシーのために情報を抽象化しても、読者視点でのアクション可能性を殺さない。

### 具体性を維持する軸
- **数値**: コスト・件数・比率など定量的な情報は省略しない
- **手段**: 何を使って・どう指示して達成したかを含める（行動主体の明示）
- **教訓**: 読者が自分の状況に重ねて「注意しよう」と思えるポイントを残す

### 判定例
```
❌「APIコストを最適化した」
  → 抽象的。読者は何も学べない

✅「$2.33/day → $0.27/dayに落ちた」
  → 数値があるので「自分のAPIコストも確認しよう」と思わせる

❌「クエリの重複に気をつけた方がいい」
  → 一般論。読者の行動を促せない

✅「機能を足したときにいきなり料金が跳ね上がるの見落としがち」
  → 読者が自分の状況に重ねて「確認しよう」と思える具体的な教訓
```

**原則: 実際にやっているかどうかは重要ではない。読者に価値あるナレッジを提供できているかが焦点。**

## BIPトピック選定の優先順位

BIP（Build in Public）は「自分の体験日記」ではない。読者への価値提供が最優先であり、自分が経験した内容に限定する必要はない。

### 優先順位
1. **情報の価値**: 読者にとって有用なナレッジか（最重要）
2. **整合性**: persona-dbのスキル・ツール・知識レベルと矛盾しないか
3. **当事者性**: 実際に経験した内容か（あればプラスだが必須ではない）

### なぜ体験縛りが有害か
- zeitgeistやバズツイート分析で得た知見・トレンドを活用できなくなる
- ネタの幅が極端に狭くなり、内輪感が出る
- 読者にとっての価値よりも「嘘をつかない」ことが優先されてしまう

### 判定例
```
❌ 自分が今日やったことだけをBIPにする
  → 「APIコスト最適化した」「スクリプト書いた」ばかりになる

✅ バズ分析やzeitgeistで見つけた有用なナレッジをBIPとして発信
  → persona-dbのスキル・知識と矛盾しない範囲で、読者に価値ある情報を提供

❌ 経験していないことを「やった」と嘘をつく
  → 嘘は禁止。ただし「〜してみた」ではなく「〜という知見がある」のフレーミングで書ける
```

**原則: BIPの価値 = 読者へのナレッジ提供。体験は信頼性のブーストであり、必須条件ではない。**

## 自己テスト（投稿前チェック）

生成したツイートに対して以下を自問する:

1. **5条件チェック**: 5つの価値条件のうち、どれを満たしているか？ → 1つも該当しないなら廃棄
2. **時間責任チェック**: 読み手の時間に見合う価値があるか？ → 「ふーん」で終わるなら不合格
3. **人格投影チェック**: この人が書いた意味があるか？ → 誰でも書ける内容なら不合格
4. **自分読みテスト**: 自分のTLに流れてきたら読みたいか？ → Noなら即廃棄
