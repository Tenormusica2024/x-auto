# フォーマット共通ルール

全ツイート種別で適用。スキル固有の上書きがある場合はそちらを優先。

## 文字数

| 種別 | 制限 |
|------|------|
| 短文ツイート | 300字以内 |
| 引用RT | 100字以内 |
| 長文ツイート | 制限なし（250-350字目安） |
| BIPツイート | 300字以内 |

### X Premium文字数制限とタイムライン表示

- X Premiumプランでは最大25,000字まで投稿可能（ベーシック: 4,000字）
- 長文ツイートの文字数制限は緩いが**タイムライン上では280字で折りたたまれ「もっと見る」表示**になる
- **最初の280字で興味を引く構成にする**ことが必須。数字・インパクトのある事実は280字以内に配置する
- 「もっと見る」の先には背景解説・利用方法・個人見解など「読み進めたくなる」補足情報を配置

## 読点（、）

- **短文ツイート / 引用RT**: 完全禁止。文の区切りは改行のみ
- **長文ツイート**: 最小限の使用は許容。基本は改行で代用

## 絶対禁止（例外なし）

- 絵文字 0個
- ハッシュタグ禁止
- デフォルトはcasual（温和な常体で統一）。politeモード指定時は敬語可（tone.jsonのformality_modes参照）
- 伸ばし棒語尾禁止（「〜だよなー」「〜かなー」）
- **日本語文中の半角スペース禁止**: 日本語テキスト同士の間に半角スペースを挟まない。AIが文節区切りとして半角スペースを入れる癖があるが、人間はやらない。英数字と日本語の間も不要（例: `CLAUDE.mdに` ← スペース不要）
  - ❌ `残す形にしたら セッション途中で`
  - ✅ `残す形にしたらセッション途中で`

## 改行・空行ルール

### 基本原則

- 1つの文・文章塊の途中での改行は完全禁止
- 短文を頻繁に改行しない（「Aか / Bか / Cだ」はインフルエンサー風で作為的）
- 読点・接続詞で自然に繋ぐ

### ブロック構成（空行の使い方）

ツイートは「意味のブロック」を**空行（Enter2回）**で区切る。行の途中での改行（Enter1回）は使わない。

```
[ブロック1: 主張 or フック or エピソード導入]
                                          ← 空行
[ブロック2: 根拠 or 具体化 or エピソード本体]
                                          ← 空行
[ブロック3: 短い締め（等身大の実感・感情）]
```

**ブロック内の折り返しは端末に任せる。** 意図的な改行をブロック内に入れない。テキストが入力欄の幅を超えて自然に折り返されるのは問題ない。

### 空行位置の決定原則（つながり優先）

**空行は「内容のつながり」で決める。行数で機械的にパターン化しない。**

- つながりのある行同士の間には空行を入れない（同じブロックとして扱う）
- 話題の転換・視点の切り替え・結論の提示の前に空行を入れる

#### 全3行の場合

最後の1行（締め）の前だけに空行を入れる:
```
[1行目: 状況・フック]
[2行目: 具体化・展開]

[3行目: 実感・締め]
```

#### 全5行以上の場合

内容のまとまりでブロックを作る。つながり優先なので行数配分は固定しない:
```
例A: 2行（空行）2行（空行）1行
例B: 3行（空行）1行（空行）1行
例C: 1行（空行）3行（空行）1行
```
どのパターンになるかは内容次第。「この行とこの行は1つの話をしている」ならブロック内に収める。

**禁止**: 全ツイートで毎回同じ改行パターン（例: 必ず1行+空行+1行+空行+1行）を繰り返すこと。パターンの固定化はAI臭さの原因。

### 締めブロックの長さ

- 締めは**短く**する（目安: 20文字以内）
- 空行の後にポンと短い1行が来ることで、結論が視覚的に際立つ
- 動詞1語だけ（「慣れません」等）は感情が強く載りすぎる場合がある。副詞1つ足す程度（「なかなか慣れません」等）で等身大感が出る

## 引用リンク

- ツイート本文の直下に必ず引用リンクを配置
- URLがない場合: 情報源を明示（「なし（[情報源の説明]）」）

## 略語禁止

サービス名・サイト名は正式名称で表記:
- ❌「HN」→ ✅「Hacker News」
- ❌「PH」→ ✅「Product Hunt」
- ❌「GH」→ ✅「GitHub」
