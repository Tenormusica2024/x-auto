# X投稿文生成スキル（バズ文体適用版） [EXPERIMENTAL]

generate-tweetの亜種。バズ文体リファレンスを参照して文面を調整するフローを追加。
テスト段階のため、常用するかは運用結果を見て判断する。

## 差分（通常版との違い）

通常の `generate-tweet/PROMPT.md` のフローをすべて実行した上で、
**ステップ4の直後**に以下のバズ文体適用ステップを追加実行する。

それ以外のステップ（ペルソナ読み込み、ネタ選定、鮮度チェック、レビュー等）は通常版と同一。

## 実行手順

### ステップ0〜4: 通常版と同一

`skills/generate-tweet/PROMPT.md` のステップ0〜4をそのまま実行。
（ステップ1.5の界隈ムードチェック含む）

### ステップ4.5: バズ文体適用（本スキル固有）

**ステップ4で初稿を作成した後、バズ文体リファレンスを参照して微調整する。**

#### 4.5-A: トピックのカテゴリ判定

作成したツイートのトピックを以下のいずれかに分類:

| カテゴリ | 判定基準 |
|---------|---------|
| AIナレッジ | Tips/ハウツー/使い方/プロンプト技法/開発ノウハウ |
| AIニュース | 新リリース/発表/業界動向/速報 |
| AIツール | 新ツール紹介/既存ツールの新機能/サービス紹介 |

#### 4.5-B: カテゴリ別バズ文体チェック

`common/buzz-style-reference.md` の該当カテゴリセクションを参照し、以下をチェック:

**全カテゴリ共通（4項目）:**
1. **冒頭フック** - 最初の1文で「自分に関係ある」と思わせているか
2. **具体物** - コピペ可能な設定値/コマンド/プロンプト片が含まれているか
3. **当事者視点** - 「自分がやった/試した」視点で書かれているか
4. **出典明示** - 情報源が示されているか

**カテゴリ別追加チェック:**

- **AIナレッジ系の場合:**
  - K2チェック: 「そのまま使える具体物」があるか（最重要）
  - K1チェック: AI名で冒頭を始められるか
  - K3チェック: 当事者の体験から入っているか

- **AIニュース系の場合:**
  - N4チェック: 原文から「衝撃フレーズ」を引用符で切り取れるか
  - N2チェック: 要点を箇条書きで整理できるか（長文版向け）
  - N5チェック: 「〜という記事/発表が〜」のメモ型で入れるか

- **AIツール系の場合:**
  - T1チェック: 「何ができるか」を1文で示せているか
  - T4チェック: 「無料」「公開」「対応」等のキーワードが自然に入るか

#### 4.5-C: 微調整

チェック結果を踏まえて文面を微調整する。

**微調整の原則:**
- 全項目を満たす必要はない。適用可能なものだけ調整
- 既存ルール（anti-ai-rules, expression-rules, format-rules）が常に優先
- バズ文体パターンと既存ルールが矛盾する場合は既存ルールに従う
- 文字数制限（140字）は厳守

**既存ルールとの矛盾解消ガイド:**
- 「損してる」「全員使え」→ 聴衆訴えかけ禁止。「自分はこれで楽になった」に変換
- 短文+多改行演出 → 改行ルール維持。段落単位で区切る
- CTA/煽り → 「自分の結果を示す」で代替

### ステップ5〜8: 通常版と同一

`skills/generate-tweet/PROMPT.md` のステップ5〜8をそのまま実行。
（ステップ7.5のDiscord #tweet-drafts自動保存を含む）
