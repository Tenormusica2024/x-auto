# 文体ラーニング・詳細言語化プロンプト

## 目的
提供されたツイートから文体を限りなくラーニングし、同じ文言や言い回しを極力使わず、価値のあるエンジニアナレッジをシェアするための文体プロンプトを作成する。

---

## ステップ1: 文体要素の詳細分解

提供されたツイートから以下の要素を抽出・分析する:

### 1-1. 構造的特徴
- 文数
- 段落構成
- 改行のタイミング
- 情報の配置順序（挨拶→本文→締め、等）

### 1-2. 語彙的特徴
- 使用する語彙のレベル（カジュアル/フォーマル/専門的）
- 漢字/ひらがな/カタカナのバランス
- 専門用語の使用頻度
- 英単語の混ぜ方

### 1-3. 文法的特徴
- 文末の形（「〜だ」「〜です」「〜である」「〜かな」等）
- 接続詞の有無・種類
- 読点（、）の有無・頻度
- 係り結びの構造

### 1-4. 表現的特徴
- 比喩の使用有無
- 体言止めの頻度
- 倒置の有無
- 繰り返しの使用
- リズム感（短文の連続/長文の積み重ね）

### 1-5. 感情的特徴
- トーン（ポジティブ/ニュートラル/分析型/批判的）
- 感情表現の有無・種類
- 読者への距離感（遠い/近い/対等）

### 1-6. 視覚的特徴
- 絵文字の有無・種類・頻度
- 改行による余白の作り方
- 記号の使用（「！」の連投等）

---

## ステップ2: パターン抽出と分類

### 2-1. 挨拶パターン
| パターン | 例 | 頻度 |
|---------|---|------|
| シンプル挨拶 | 「おはようございます！」 | 高 |
| 曜日付き | 「おはようございます。金曜日。」 | 中 |
| 天気・気分付き | 「おはようございます。今日は寒いですね。」 | 中 |

### 2-2. 文展開パターン
- 挨拶のみ
- 挨拶 + 今日のテーマ
- 挨拶 + 昨日の活動 + 感想
- 挨拶 + 疑問形

### 2-3. 締めパターン
- 締めなし（文末で終了）
- 「よろしくお願いします」系
- 感想で締める
- 疑問で締める

---

## ステップ3: マイクロスコープ分析（各要素の言語化）

### 3-1. 文末1つ1つの意味を分析
「おはようございます！」の「！」は何を意図しているか
- 単なる区切り
- 明るさの演出
- 読者への呼びかけ

### 3-2. 語彙選択の背景
なぜ「調子進めていきます」なのか
- 「頑張ります」との違い
- 「よろしくお願いします」との違い
- エンジニアとしてのアイデンティティ

### 3-3. 改行1つ1つの意図
なぜここで改行しているのか
- 視覚的な余白
- 意味の区切り
- リズム作り

---

## ステップ4: バリエーション生成ルール

### 4-1. 毎日変化をつけるためのルール
- 構造のローテーション（シンプル→曜日付き→昨日の活動→疑问形）
- 語彙のローテーション（同義語の使い分け）
- 文末のローテーション（「です」「だ」「でしょう」の使い分け）

### 4-2. 価値あるエンジニアナレッジの定義
- 具体的な技術情報を含む
- 経験則に基づく知見
- 検証・実験の結果
- 失敗談と学び

### 4-3. 同じ文言を避けるルール
- 直近N件のツイートで同じ言い回しを使わない
- 類似表現でも語順を変える
- 視点を変える（自分視点→読者視点→第三者視点）

---

## ステップ5: 文体再現プロンプト作成

上記の分析に基づいて、以下のプロンプトを作成する:

```
# 朝ツイート生成プロンプト（文体ラーニング版）

## 基本文体
[ここにステップ1-4の分析結果を反映]

## バリエーションルール
[ステップ4のルールを反映]

## 禁止パターン
- 同一文構造の連続使用
- 同一文末の連続使用
- 同一語彙の連続使用
- 同一改行パターンの連続使用
```

## 出力形式

### 5-1. 文体分析レポート
```
## 文体分析結果

### 構造的特徴
- 文数: [平均/最少/最多]
- 段落構成: [詳細]

### 語彙的特徴
- 語彙レベル: [カジュアル/フォーマル/専門的]
- 漢字:ひらがな:カタカナ = [比率]

### 文法的特徴
- 主な文末: [例1, 例2, 例3]
- 接続詞: [有無/主なもの]

### 表現的特徴
- 比喩: [有無/例]
- 体言止め: [頻度]

### 感情的特徴
- 基調トーン: [詳細]
- 読者距離: [詳細]

### 視覚的特徴
- 絵文字: [有無/種類]
- 改行: [パターン]
```

### 5-2. バリエーションルール
```
## バリエーション生成ルール

### 構造ローテーション
1. シンプル挨拶
2. 曜日/天候/気分
3. 昨日の活動 + 感想
4. 疑問形
5. トピック言及

### 文末ローテーション
- パターンA: 「おはようございます。」で終了
- パターンB: 感想で締める
- パターンC: 疑問で締める

### 語彙ローテーション
- 同義語リスト作成
- 毎回異なる語彙を選択
```

### 5-3. 文体再現プロンプト
```
# [タイトル]

## 基本文体
[詳細な文体指定]

## バリエーションルール
[詳細なルール]

## 禁止パターン
[禁止事項リスト]
```

---

## 注意事項
- 提供されたツイートからのみ学習する（外部情報の混入禁止）
- 分析は客観的かつ詳細に行う
- 「なんとなく」ではなく「なぜそうなるか」まで言語化する
- バリエーションは実用的かつ無理のない範囲で
