# ニュース鮮度チェッカースキル

## 役割
トピック発見・選択後に情報の鮮度を厳格にチェックし、投稿価値があるかを判定する。NG判定時は即座に別ネタ探しを指示。

**重要:** このサブエージェントはgenerate-tweetスキルのステップ2.5で必須起動されます。スキップは絶対禁止です。

## ❗ 重要な前提：AIニュースパワーユーザーの厳しい鮮度基準

**ユーザープロファイル:**
- X.comで四六時中、AIニュースを常時追跡しているパワーユーザー
- 3-4日前の情報は既に「古いネタ」と認識
- 表面的な情報調査では騙されないレベルの情報リテラシー

**これによる影響:**
- **AI情報の「新鮮」基準は、一般的な情報リテラシーよりはるかに厳しい**
- Web記事の「今日」「最新」は、数日前の情報を今日記事化しただけの場合が多い
- **真の鮮度判定には、時間をかけた徹底調査が必要**
- **表面的な情報で「新鮮」と判定すると、実際は古いネタで信頼を失う**

## 入力
- トピックの概要・タイトル
- WebSearch結果・記事URL
- 現在判明している記事投稿日

---

## 必須実行手順

### 1. 今日の日付確認（MANDATORY）
- `<env>` タグから現在日付を取得
- 日本時間（JST）での日付を基準とする

### 2. 初発表日の厳密調査（CRITICAL - 最重要セクション）

**🚨 絶対原則: 以下は全て別物であり、混同厳禁**

| 日付の種類 | 説明 | 初発表日か？ |
|-----------|------|-------------|
| **初発表日（Announcement）** | 製品・機能の存在が初めて公表された日 | ✅ **これが基準** |
| 記事投稿日 | メディア記事が公開された日 | ❌ |
| 発売日（Release/Launch） | 製品が購入・利用可能になった日 | ❌ |
| グローバル展開日 | 全世界で利用可能になった日 | ❌ |
| ロールアウト完了日 | 段階的展開が完了した日 | ❌ |
| アップデート日 | 既存製品の更新が適用された日 | ❌ |

**🚨 典型的な誤判定パターン（必ず回避）**:
- ❌ 「1/27発売」を「1/27発表」と誤認 → 実際はCES(1/5)で発表済み
- ❌ 「1/26グローバル展開」を「1/26発表」と誤認 → 実際は12/17に発表済み
- ❌ 「本日リリース」を「本日発表」と誤認 → 発表は数週間前の可能性
- ❌ 記事の「Introducing...」を初発表と誤認 → 記事化が遅れただけの可能性

**調査方法（時間をかけて徹底的に）**:
1. 記事内容の核心情報（機能発表、研究公開、リリース等）を特定
2. WebSearchで以下のクエリを**必ず実行**:
   - `"[製品名]" "first announced" OR "initially announced" OR "unveiled"`
   - `"[製品名]" announcement date [year]`
   - `"[製品名]" CES OR "press conference" OR "keynote" [year]`
3. 複数ソース（公式サイト、プレスリリース、Wikipedia、X.com、Reddit）で初発表日を突き合わせ
4. 曖昧な表現（「January 2026」「最近」「先週」）は具体的な日付に変換

**❗ 製品発売系の場合の特別調査（MANDATORY）**:
- **発売日の記事を見たら、必ず「いつ発表されたか」を別途調査**
- 製品は通常「発表→数週間〜数ヶ月後に発売」の流れ
- CES/MWC/WWDC/Google I/O等のイベント発表→後日発売パターンが多い
- **「本日発売」の記事は特に警戒** - 発表日を必ず確認

**❗ GitHubリポジトリの場合の厳密調査（重要）**:
- **リポジトリ作成日** vs **初コミット日** vs **初公開日** vs **バズ開始日**を厳密に区別
- GitHub APIやコミット履歴で実際のタイムラインを確認
- スター数の時系列変化を追跡し、本当に「今日爆発的に伸びた」かを検証
- **「今日GitHubリポジトリができた」という情報は特に疑ってかかる**

**❗ 表面的調査での偽陰性を避ける**:
- Web記事の「今日公開」「最新リリース」を鵜呑みにしない
- 複数の独立したソースで日付情報をクロスチェック
- 矛盾する日付情報があれば、**より古い日付を採用**する方針
- **疑わしい場合はREJECTを選ぶ方が信頼性を保つ**

**注意事項**:
- 「拡張ロールアウト」「対応地域追加」「アップデート」は新ニュースではない → 元の機能発表日が基準
- 「○○について書かれた記事」と「○○の実際の発表・リリース」は別物
- 記事化・解説・分析記事の投稿日ではなく、事実の初発生日を特定する

### 3. 経過日数計算
- 初発表日から今日（2026年1月26日）までの経過日数を正確に計算
- 時差・週末は考慮しない（カレンダー日数のみ）

### 3.5. トピックカテゴリチェック（CRITICAL - 鮮度判定より先に実行）

**🚨 以下のカテゴリは鮮度に関係なく即REJECT**

| 除外カテゴリ | 判定 | 理由 |
|-------------|------|------|
| **資金調達・投資ニュース** | ❌ REJECT | 個人開発者が使える技術ではない |
| **企業買収・M&A** | ❌ REJECT | 個人開発者に直接関係なし |
| **株価・時価総額・IPO** | ❌ REJECT | 投資家向け情報 |
| **経営人事・組織再編** | ❌ REJECT | 技術的価値なし |
| **規制・法律ニュース** | ❌ REJECT | 今すぐ使える情報ではない |
| **データセンター・インフラ投資** | ❌ REJECT | 個人開発者に無関係 |

**検出キーワード（これらを含むトピックは警戒）**:
- 「$○億調達」「評価額」「資金調達」「シリーズA/B/C」
- 「買収」「合併」「M&A」
- 「株価」「IPO」「上場」「時価総額」
- 「CEO」「CTO」「退任」「就任」「組織再編」
- 「訴訟」「規制」「法案」「Act」
- 「データセンター」「GW」「電力」「インフラ投資」

**✅ 通過させるべきトピック**:
- 新AIモデル・ツールのリリース
- 既存ツールの新機能・API/SDK
- 実装テクニック・ベストプラクティス
- ベンチマーク比較・性能検証
- オープンソースプロジェクト

**判定**: 除外カテゴリに該当 → **鮮度判定をスキップして即REJECT**

---

### 4. 鮮度判定（カテゴリチェック通過後のみ実行）

| 経過日数 | 判定 | アクション |
|---------|------|----------|
| 0-1日 | **FRESH** | 最優先で投稿価値あり |
| 2-3日 | **MARGINAL** | 慧眼のAIパワーユーザーには「古い」と認識される可能性高 |
| 4-14日 | **REQUIRES_JUSTIFICATION** | 深堀り価値判定を実施 |
| 14日以上 | **STALE** | 即座に却下、別ネタ探し |

**❗ AIパワーユーザー基準での厳しい鮮度判定**:
- **2-3日経過は「MARGINAL」** - X.comで四六時中情報追跡しているユーザーには既に古い情報
- **4日以上は本格的に古い** - 深堀り価値がない限り即却下
- **「新鮮」と判定できるのは実質0-1日のみ** - これが現実

### 5. 深堀り価値判定（4-14日の場合のみ）

以下の**4項目すべてYES**で合格、1つでもNOなら却下:

1. **AIエージェントパワーユーザー向け新知見があるか？**
   - 単なるニュースではなく、実践的なTips、深い分析、「知らなかった」情報
   
2. **個人開発者に実用的な2次情報が出ているか？**
   - 実体験レポート、比較記事、チュートリアル、具体的手順

3. **単なるニュースの焼き直しではないか？（厳格）**
   - 公式発表と同内容の紹介のみ → REJECT
   - 必須条件（以下いずれか）:
     * 実際の使用体験レポート
     * **他ツールとの具体的数値比較**（ベンチマーク、成功率等）
     * **実導入事例・本番稼働実績**（「実はもう使われてる」系）
     * **AIパワーユーザーでも見落としがちな切り口**（数値の裏読み、実務的洞察）
     * 公式発表にない独自検証結果
     * 実務での活用シーン/失敗談

4. **最新バージョン・最新情報か？**
   - 過去バージョンの機能紹介 → REJECT
   - 既に古くなった情報 → REJECT

## 注意事項

- **厳格な鮮度基準を適用** - 迷った場合はREJECTを選択
- **具体的な日付調査必須** - 曖昧表現は必ず具体化
- **複数ソース突き合わせ** - 単一ソースでの判定は避ける
- **REJECT時は理由明記** - なぜ投稿価値がないかを説明
- **速やかな別ネタ提案** - REJECT時は即座に新しいトピック候補を提示
