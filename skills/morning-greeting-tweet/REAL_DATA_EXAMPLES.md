## 🚀 優先度: 高 - AI系トラブルシューティング

### 例1-1: BashOutputポーリングの無限ループ問題
```
おはようございます。昨日、BashOutputポーリングでstatus:runningが延々続く問題を調べてた。
Claude Codeの内部状態異常だったから再起動で解決しました。腹落ちしました。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: BashOutputポーリングで `status: running` が延々と続く
- 原因: Claude Code Bashセッションの内部状態異常
- 解決策: Claude Codeの再起動（VSCode再起動 or `claude` コマンド再実行）

**技術的価値:** AI開発環境のトラブルシューティング、Bashセッション管理

### 例1-2: GitHub認証エラーの根本原因
```
おはよう。昨日、GitHub CLIでHTTP 401認証エラーが出る問題を調べてた。
環境変数GITHUB_TOKENがkeyring認証をオーバーライドしてたのが原因でした。道筋が見えました。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: GitHub CLIでHTTP 401認証エラー
- 原因: 環境変数 `GITHUB_TOKEN` がkeyring認証をオーバーライド
- 解決策: `Remove-Item Env:\GITHUB_TOKEN` で環境変数削除

**技術的価値:** GitHub CLI認証、環境変数優先順位の理解

### 例1-3: セマンティック検索の評価妥当性问题
```
おはようございます。昨日、セマンティック検索のカバー率31%での評価が妥当か検証してた。
インデックス未済みファイルがあると評価が不公平になることが分かりました。方向性は合ってそうです。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: インデックスカバー率31%での評価の妥当性
- 原因: agent-browser.mdがセマンティック圏外だがGrepでは確実ヒット
- 解決策: 完全インデックス化で改善、または二段階検索でフォロー

**技術的価値:** セマンティック検索、インデックス設計、評価手法

### 例1-4: 経験者ほどAIで遅くなる逆説
```
おはよー。昨日、経験豊富な開発者ほどAIツールで19%遅くなるデータを分析してた。
AIの提案を過度に検証しちゃうのが原因だと分かりました。知見が増えました。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: 経験豊富な開発者はAIツール使用時に19%遅くなる
- 原因: AIの提案を過度に検証・修正してしまう
- 解決策: 「AIを信頼する場面」「疑う場面」の判断力を磨く

**技術的価値:** AIツールの生産性、使い分け判断、開発者心理学

### 例1-5: 二段階検索で精度改善
```
おはようございます。昨日、セマンティック検索に二段階フィルターを追加してた。
セマンティックで候補を広げてからGrepで絞り込む形式にしたらHit@5が改善しました。着地しました。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: セマンティック単独では比較系・抽象テーマ系が失敗
- 解決策: セマンティック（広げ）→ Grep（絞り）→ 時系列Rerank
- 結果: Hit@5 60% → 改善期待

**技術的価値:** ハイブリッド検索、精度改善、検索アーキテクチャ

## 🔧 優先度: 中 - クラウド・開発環境系

### 例2-1: Cloud Runキャッシュ問題の根本原因
```
おはよう。昨日、Cloud Runで変更が反映されない問題を調べてた。
ドキュメントに固定ディレクトリパスが記載されてたのが原因でした。ナンスですけど解決しました。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: デプロイ後にコード変更が反映されない
- 原因: CLAUDE.mdに固定ディレクトリパスが記載され、誤ったディレクトリからデプロイ
- 解決策: 固定パス記載を禁止、都度ユーザー確認フローを追加

**技術的価値:** ドキュメント管理、デプロイ自動化、キャッシュ対策

### 例2-2: nulファイルエラーのGit Bash制限
```
おはようございます。昨日、Bashコマンドでnul: cannot overwriteエラーが出る問題を調べてた。
Git Bash環境でnulが特殊ファイル扱いされるのが原因でした。PowerShellに移行してクリアです。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: `dir ... 2>nul` 等のWindows構文でエラー
- 原因: Git Bash環境で `nul` が特殊ファイルとして扱われる
- 解決策: PowerShellを使用 `powershell -Command "..."`

**技術的価値:** Windows環境差異、Bash制限、PowerShell移行

### 例2-3: Claude Codeフックのビープ音問題
```
おはよー。昨日、Claude Code回答完了時にビープ音が鳴る問題を調べてた。
VOICEVOX未起動時にzundamon_speak.jsがエラー出してるのが原因でした。対処しました。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: Claude Code回答完了時のSEと同時にビープ音が複数回鳴る
- 原因: SubagentStopフックの `zundamon_speak.js` がVOICEVOX未起動時にエラー
- 解決策: settings.jsonからSubagentStopフックを削除

**技術的価値:** Claude Codeフック、エラーハンドリング、VOICEVOX連携

### 例3-1: OSダイアログ検出の限界
```
おはようございます。昨日、OSネイティブダイアログの開閉状態検知問題を調べてた。

```

**元ネタ:**
- 問題: Claude in ChromeがOSネイティブダイアログの開閉状態を検知できない
- 解決策: クリップボード方式（Ctrl+V）に移行

### 例3-2: タイミングバグの検出
```
おはよう。昨日、自動入力スクリプトのタイミング問題を調べてた。
実行が早すぎて別のウィンドウに入力されちゃうバグ見つけました。パターンが見えました。
本日もよろしくお願いします。
```

**元ネタ:**
- 問題: dialog_filler.pyの実行タイミングが早すぎて、記事本文が壊れる
- 原因: OSダイアログが開く前にスクリプト実行
- 解決策: delayを2.0〜2.5秒に設定、またはクリップボード方式に移行
