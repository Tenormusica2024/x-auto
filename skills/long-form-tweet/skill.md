# long-form-tweet

長文ツイート生成スキル（文字数制限なし版）

## 概要

140字制限を超える長文ツイートを生成する。
新ツール紹介・技術解説・深掘り分析など、情報量が必要な場面で使用。

## 起動条件

以下のキーワードで自動起動:
- 「長文ツイート」「長めのツイート」
- 「詳しく紹介するツイート」
- 「文字数気にしないで」
- `/long-form-tweet`

long-form-tweet/

## 基本スタンス

### 新ツール紹介時のデフォルトトーン

**肯定的・建設的なトーンをデフォルトとする**

| 項目 | デフォルト | 例外ケース |
|------|----------|----------|
| 新ツール紹介 | 肯定的に紹介 | 明らかな問題がある場合のみ注意喚起 |
| 技術解説 | 実用性を強調 | セキュリティリスク等は正直に |
| 比較記事 | 各ツールの強みを紹介 | 致命的欠陥は指摘 |

**理由:**
- ネガティブ情報はすぐ広まるが、ポジティブ情報は埋もれやすい
- 「試してみたい」と思わせることがエンゲージメントに繋がる
- 批判は誰でもできるが、価値を見出すには知識が必要

## 長文ツイートの構成ルール

### 推奨構成（6段階フロー）

```
1. 導入フック（1-2行）
   └─ ポジティブな第一印象を設定

2. 技術的仕組み（2-3行）
   └─ どう動くかを具体的に説明

3. 自然言語での例示（2-3行）
   └─ 専門用語を避けた具体例

4. 実用的メリット（2-3行）
   └─ 即座に得られる利点

5. システム的優位性（2-3行）
   └─ 長期的・構造的な強み

6. ターゲット適合性（1-2行）
   └─ 誰に向いているかで締める
```

### 各段階の詳細

#### 1. 導入フック
**目的:** 読者の興味を引き、ポジティブなトーンを設定

**パターン例:**
- 「[ツール名]が思った以上に実用的で」
- 「[ツール名]の組み合わせが面白くて」
- 「[ツール名]試してみたけど」

**禁止:**
- ❌ 「[ツール名]がヤバい」（抽象的すぎる）
- ❌ 「[ツール名]について調べてみた」（受動的）

#### 2. 技術的仕組み
**目的:** どう動くかを具体的に説明し、信頼性を確保

**必須要素:**
- 内部処理の概要（何が何に変換されるか）
- 技術スタックの言及（フレームワーク、言語等）

**例:**
```
補間関数がフレーム番号から位置や透明度を計算する形でアニメーション組める
```

#### 3. 自然言語での例示
**目的:** 専門家以外にも「自分でもできそう」と思わせる

**必須要素:**
- 実際のプロンプト例（カギカッコで囲む）
- 専門用語を日常語に変換

**変換例:**
| 専門用語 | 自然言語 |
|---------|---------|
| イージング | ふわっと現れてスッと止まる感じ |
| トランジション | 場面が切り替わるとき |
| インターポレーション | なめらかに変化する |

#### 4. 実用的メリット
**目的:** 即座に得られる具体的な利点を示す

**パターン:**
- 「〜の知識なくても〜で通じる」
- 「〜するだけで〜できる」

#### 5. システム的優位性
**目的:** 長期的・構造的な強みを示す

**例:**
- 再現性（同じプロンプトで同じ出力）
- スケーラビリティ（バッチ生成可能）
- エコシステム統合（既存資産の流用）

#### 6. ターゲット適合性
**目的:** 誰に向いているかを明示して締める

**パターン:**
- 「〜経験ある人には馴染みやすい構成になってる」
- 「〜やってた人が〜に入る障壁が下がった感じがする」

## 文字数ガイドライン

| 項目 | 目安 |
|------|------|
| 全体 | 250-350字 |
| 各段落 | 40-80字 |
| 1行 | 制限なし（自然な区切りで改行） |

**注意:** 長文でも「1つの文章塊の途中での改行は禁止」は維持

## 品質チェックリスト

### 必須要件
- [ ] 導入がポジティブなトーンで始まっている
- [ ] 技術的仕組みが具体的に説明されている
- [ ] 専門用語に自然言語の言い換えがある
- [ ] 引用リンクが付いている

### 推奨要件
- [ ] 実際のプロンプト例がカギカッコで示されている
- [ ] ターゲット層が明示されている
- [ ] 「〜んだよね」「〜ではある」等のペルソナ口調が使われている

### 禁止事項
- [ ] 絵文字なし
- [ ] ハッシュタグなし
- [ ] 敬語なし（常体で統一）
- [ ] 「すごい」「ヤバい」等の抽象的感想なし

## 完成形サンプル

### 新ツール紹介（肯定的パターン）

```
Remotion + Claude Codeの組み合わせが思った以上に実用的で
補間関数がフレーム番号から位置や透明度を計算する形でアニメーション組める

Claude Codeに「ふわっと現れてスッと止まる感じで」って言えば
そのままinterpolate()に変換してくれるから
アニメーションの知識なくても動きのイメージを伝えるだけで通じる

しかもAIが映像じゃなくてTypeScriptを生成する仕組みなので
同じプロンプトで同じ出力が出るし
パラメータ変えてバッチ生成もできる

Reactのエコシステムがそのまま使えるから
Tailwindも既存のコンポーネントも流用できて
Webフロント経験ある人には馴染みやすい構成になってる
```

**構成分解:**
1. 導入フック: 「思った以上に実用的で」
2. 技術的仕組み: 補間関数の説明
3. 自然言語例示: 「ふわっと現れてスッと止まる感じで」
4. 実用的メリット: 知識なくても通じる
5. システム的優位性: コード生成による再現性・バッチ生成
6. ターゲット適合性: Webフロント経験者向け
