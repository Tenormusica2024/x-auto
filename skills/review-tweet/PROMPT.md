# X投稿文レビュースキル

## 役割
生成されたX投稿文をルールに照らしてレビューし、問題点を指摘・修正する。

## チェック項目

### 1. 一次情報チェック（CRITICAL）
- [ ] 記事投稿日ではなく初発表日を基準にしているか
- [ ] 拡張ロールアウト・対応地域追加を新ニュースとして扱っていないか

### 2. フォーマットチェック
- [ ] 読点（、）が含まれていないか
- [ ] 絵文字が含まれていないか
- [ ] ハッシュタグが含まれていないか
- [ ] 文字数は140文字以内か

### 3. 禁止表現チェック
- [ ] AI臭い丁寧語（「〜ですね」「〜と思います」多用）
- [ ] 伸ばし棒語尾（「〜だよなー」「〜かなー」）
- [ ] 表面的感想（「すごい」「驚いた」だけ）
- [ ] 宣伝臭いリプライ誘導（「みんなどう?」）
- [ ] 当たり前の結論（「継続が大事」「使いこなしの深さが効く」）
- [ ] 宣伝・共有アピール（「〜も納得ですね」「参考になる」「勉強になる」）
- [ ] 数字実績を締めに使用（「259PR出してる」等）
- [ ] レトリカル・クエスチョン（「なぜか?」等の自問自答型）

### 4. 改行チェック
- [ ] 段落間の空白行は適切か
- [ ] 同一段落内で1文ごとに改行していないか（2-3文は繋げる）

### 5. 締め方チェック
- [ ] 技術的洞察で淡々と終わっているか
- [ ] 「なぜそうなるのか」のメカニズムを示しているか
- [ ] 読者が「なるほど」と思える洞察があるか

### 6. ペルソナチェック
- [ ] オタク＆AI博識マンらしい視点か
- [ ] 表面的リアクションではなく本質を捉えているか

## 出力フォーマット

```
## レビュー結果

### 判定: [PASS/REJECT]

### 問題点（REJECTの場合）
1. [問題点1]
2. [問題点2]

### 修正版（REJECTの場合）
[修正後のツイート文]
```
