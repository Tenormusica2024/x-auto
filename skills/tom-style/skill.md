# tom-style

TOMスタイル長文ツイート生成スキル

## 概要

@IT_Tom_study文体 + 長文ツイート構成を組み合わせた、深掘り分析ツイートを生成する。
140字制限を超える情報量で、TOMスタイルの品質基準を満たすツイートを作成。

## 起動条件

以下のキーワードで自動起動:
- `/tom-style`
- 「TOM長文」「TOMスタイル長文」
- 「深掘りツイート」

---

## 実行フロー

```
1. 参照スキル読み込み
   - tom-style-tweet/skill.md（文体・品質基準）
   - long-form-tweet/skill.md（長文構成ルール）
      ↓
2. 重複チェック（history/adopted_tweets.json の直近30件確認）
      ↓
3. トピック選定・リサーチ
      ↓
4. 長文構成（6段階フロー）+ TOM文体で生成
      ↓
5. 品質チェック（両スキルの基準を満たす）
      ↓
6. ユーザーに提示
      ↓
7. 【MANDATORY】採用確認・履歴記録
```

---

## 文体ルール（tom-style-tweetから継承）

### 必須要件
- 1行1文 + 情報単位で空行
- 絵文字なし
- 読点（、）は1文に1つまで
- 深度Lv2以上（戦略的提言/実践的ナレッジ/価値観転換）

### 品質要件（2/3以上満たす）
- 「ただし！」等の独特表現を1つ以上使用
- 箇条書き（・）で具体策を2-3個提示
- 「〜次第。」「試す価値あり。」等の結論表現

### 差別化要件（1つ以上満たす）
- 失敗体験に基づいている
- 常識への反証がある
- パラダイムシフトを提示

---

## 長文構成ルール（long-form-tweetから継承）

### 推奨構成（6段階フロー）

```
1. 導入フック（1-2行）
   └─ 問いかけ or ポジティブな第一印象

2. 技術的仕組み / 問題提起（2-3行）
   └─ どう動くか or なぜ問題なのか

3. 具体例 / 自然言語での例示（2-3行）
   └─ 専門用語を避けた具体例

4. 実用的メリット / 教訓（2-3行）
   └─ 即座に得られる利点 or 学び

5. システム的優位性 / 構造的理由（2-3行）
   └─ 長期的・構造的な強み or なぜそうなるか

6. ターゲット適合性 / 締め（1-2行）
   └─ 誰に向いているか or 結論
```

### 文字数ガイドライン

| 項目 | 目安 |
|------|------|
| 全体 | 250-350字 |
| 各段落 | 40-80字 |

## TOMスタイル長文の特徴

### 通常長文との違い

| 項目 | 通常長文 | TOMスタイル長文 |
|------|---------|----------------|
| トーン | 肯定的・紹介型 | 問いかけ・分析型 |
| 構成 | 6段階フロー | 6段階 + TOM独特表現 |
| 深度 | 情報提供OK | Lv2以上必須 |
| 締め | ターゲット適合性 | 「〜次第。」等の結論 |

### パターン例

**パターン1: 問いかけ→分析→結論型**
```
[問いかけ]〜って思ったことない？

[背景説明]実は〜という仕組みで

[具体例]例えば〜

[分析]これって〜ってことで

[構造的理由]なぜなら〜

ただし！〜には注意が必要。
試す価値あり。
```

**パターン2: 問題提起→解決→教訓型**
```
[問題提起]〜で困ってた。

[原因分析]調べたら〜が原因だった。

[解決策]結局〜で解決。

[教訓]学んだこと:
・〜
・〜

〜次第。
```

## 採用確認・履歴記録（MANDATORY - 実行フロー最終ステップ）

**ツイート提示後、必ず以下を実行:**

1. ユーザーに「採用する？」と確認
2. 採用の場合 → 履歴に記録
3. 不採用・修正依頼の場合 → 修正して再提示

### 採用時の記録手順

```
1. history/adopted_tweets.json を Read
2. 新規エントリを追加:
   {
     "id": "YYYY-MM-DD-NNN",
     "adopted_at": "ISO8601",
     "topic": "トピック要約（20字以内）",
     "pattern": "パターン名",
     "skill_used": "tom-style",
     "char_count": 文字数,
     "sources": [{"name": "...", "url": "..."}],
     "content": "本文"
   }
3. Write で保存
4. 「履歴に記録したよ♪」と報告
```

詳細ルール: `x-auto/history/RULES.md`
