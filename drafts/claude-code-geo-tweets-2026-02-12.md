## Tweet 1: Claude Code定期実行 x GEO鮮度シグナル

Claude Codeはclaude -pで非対話実行できるからTask Schedulerに登録すると毎朝勝手に動く。AIニュース収集や記事ネタの構造化を寝てる間にやらせてる

AI検索(ChatGPT/Perplexity)に引用されるには「定期更新される構造化データ」が必要で手動だと絶対続かない。Claude Codeの定期自動実行はGEO対策と相性が良すぎると思うんだよね。ChatGPTもCursorもこの「放置で勝手に動く」ができない

## Tweet 3: Hooks x Build in Public自動化

Claude CodeにはHooksっていう「特定イベント時に自動スクリプト実行」の仕組みがある。git commitしたらBuild in Publicツイートのドラフト自動生成とか。Build in Publicの最大の敵は「継続できない」ことで仕組みで殴るのが正解だと思ってる

開発ログが構造化されて自動蓄積されるからAI検索に引用される一次データが勝手に貯まる。「続けられる仕組み」を最初に作れるのがClaude Codeの強み

## Tweet 4: 一次データ自動蓄積 x GEO最適化

AI検索時代のSEO(GEO)で一番強いのは「AIが生成できない一次データ」を持ってること。具体的な数値や運用ログや実装過程の記録

Claude Codeは開発過程そのものを記録しながら進むからBuild in Publicの素材が自動で溜まる。--resumeでセッション文脈を保持したまま数日間の作業ログが残るのも地味に強い。「実際にやってみた」系コンテンツが最強なのはAIには体験が書けないから
