## Tweet 1: CLAUDE.md（/init で自動生成）

Claude Codeで /init って打つとCLAUDE.mdってファイルが自動生成される
これがプロジェクトの取扱説明書みたいなもので「TypeScriptで書いてる」「テストはpytestで走らせる」「コミットは日本語で」みたいなルールを自由に書ける
一度書いておけばセッション跨いでもClaude Codeがそのルール守って作業してくれるから毎回同じ指示を繰り返さなくていい
ソロ開発でも「1週間前の自分」に引き継ぎしてる感覚になるんだよね

## Tweet 2: Plan Mode（Shift+Tab）

Claude Codeでいきなり「実装して」って投げるよりShift+TabでPlan Modeに切り替えてから指示出すと結果がだいぶ変わる
Plan Modeだとまずコードベース全体を調べてから実装計画を出してくれて「この方針でいい?」って聞いてくる
承認してから実装に入るからやり直しが減るし方向性のズレも起きにくい
複雑なタスクほど効果出るからとりあえずShift+Tab押す癖つけるといいと思うんだよね

Claude Codeのhooksって機能が地味に強くて特定のタイミングで自動的にコマンドを走らせられる

## Tweet 4: claude --resume（セッション復元）

Claude Codeのセッションが途中で切れたり翌日に続きやりたい時は claude --resume で前回のセッションから再開できる
コンテキストがそのまま引き継がれるから「昨日の続きで」って一から説明し直さなくていい
ターミナル閉じても数日前のセッションでも復元できる
これ知らないと毎回コンテキスト再構築するハメになるから早めに覚えておいたほうがいいと思うんだよね

**ポイント**: 「セッション切れたら最初から」と思ってる人が多い。復元できることを知るだけで体験が変わる

## Tweet 5: スラッシュコマンド（.claude/commands/）

Claude Codeで毎回同じような指示出してるなら .claude/commands/ にMarkdownファイル置くだけでスラッシュコマンドにできる
「コードレビューして」を /review で一発実行とか「デプロイ手順」を /deploy にしておくとか
プロンプト内に $ARGUMENTS って書けば引数も渡せる
自分専用のCLIツール作ってる感覚に近くてプロンプトをコード資産として管理できるのが面白いんだよね

なし（Claude Code カスタムスラッシュコマンド）

---
**文字数**: 約240字
**種別**: 長文ツイート
**ポイント**: 「Markdownファイル置くだけ」の手軽さが非エンジニアにも響く。$ARGUMENTSは上級者向けだが存在を知るだけで可能性が広がる
